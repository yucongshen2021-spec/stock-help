### 11:35 全面联动测试 + 3 个严重后端 Bug 修复

**测试范围**：6 个后端 REST API、SSE 聊天流、股票代码/名称识别、UI 完整交互。

**修复的严重 Bug：**

1. **AKShare period 参数 API 变更**
   - 文件: `backend/app/services/stock_data.py`
   - 原因: `akshare 1.18.63` 的 `stock_zh_a_hist` 已将 `period` 参数从中文 `"日k"/"周k"/"月k"` 改为英文 `"daily"/"weekly"/"monthly"`，导致 K 线/指标/技术分析 3 个 API 全部 500 错误。
   - 修复: 改用英文 period 参数。

2. **AKShare 默认 User-Agent 被东方财富屏蔽**
   - 文件: `backend/app/services/stock_data.py`
   - 原因: `requests` 库默认 UA `python-requests/2.x` 被 `quote.eastmoney.com` 主动关闭连接 (`RemoteDisconnected`)，导致 `stock_zh_a_spot_em`、`stock_zh_a_hist`、`stock_bid_ask_em` 间歇失败，行情/K线/分析全部不稳定。
   - 修复: 抛弃 akshare 内部 HTTP 调用，**直接** 用 `requests + 真实浏览器 UA` 调东方财富 `push2.eastmoney.com/api/qt/stock/get` 和 `push2his.eastmoney.com/api/qt/stock/kline/get`，附带 `Referer` 头。新闻 (`stock_news_em`) 和股票列表 (`stock_info_a_code_name`) 仍走 akshare（这两个未被屏蔽）。
   - 同时加入：股票列表内存缓存 (`_STOCK_LIST_CACHE`)，避免每次聊天都下载 16 chunk CDN 文件。

3. **股票代码正则把中文当 word character 导致漏匹配**
   - 文件: `backend/app/services/stock_data.py`
   - 原因: 原正则 `\b(\d{6})\b` 在 Python 3 中失效。Python `re` 把 `\u4e00-\u9fa5` 视为 word character，所以 `"分析一下600519最近走势"` 里 "下"-"6" 之间和 "9"-"最" 之间都没有 `\b` 边界，`600519` 完全提取不出。**导致用户用代码提问时，AI 完全拿不到实时数据，只能凭训练知识胡说**（实测 AI 回答"无法获取实时数据"）。
   - 修复: 改用负向前后查 `(?<!\d)(\d{6})(?!\d)`，对中英标点都正确工作。

4. **中文股票名提取贪婪正则匹配失败**
   - 文件: `backend/app/services/context.py` + `stock_data.py`
   - 原因: 原 `_CHINESE_STOCK_PATTERN = r"[\u4e00-\u9fa5]{2,6}..."` 贪婪匹配 6 字，把 "贵州茅台最近" 当成候选去 `resolve_stock_name`，永远查不到。导致用户用股票中文名（不带代码）提问时 AI 也拿不到数据。
   - 修复: 新增 `find_stock_names_in_text`，反向扫描：遍历股票名列表，找出哪些名字是用户消息的子串。删除失效的 `_CHINESE_STOCK_PATTERN`。

### 15:30 UI 主题适配修复 — 3 个 Bug + 全局 UI 规范化

**修复的 Bug：**

1. **themeStore.ts — setTheme 缺 set() 调用**
   - 文件: `frontend/src/stores/themeStore.ts`
   - 原因: `setTheme` action 只 `return { theme }` 而没有调用 `set()`，zustand state 不会更新。
   - 修复: 改为 `set({ theme })`。

2. **AppLayout.tsx — 侧边栏 onToggle 只能关不能开**
   - 文件: `frontend/src/components/Layout/AppLayout.tsx`
   - 原因: `onToggle={() => setSidebarOpen(false)}` 永远设为 false。
   - 修复: 改为 `onToggle={() => setSidebarOpen((prev) => !prev)}`。

3. **KLineChart.tsx — ECharts 配色硬编码深色**
   - 文件: `frontend/src/components/Stock/KLineChart.tsx`
   - 原因: 所有 ECharts option 颜色（轴、网格线、tooltip 背景/文字、标签）全部硬编码为深色主题值（`rgba(255,255,255,...)`、`#6e6e6e`、`#252525` 等），亮色模式下图表元素几乎不可见。
   - 修复: 通过 `useThemeStore` 获取当前主题，`useMemo` 动态生成 ECharts option，深色/亮色各一套配色；同时周期选择器也改用 CSS 变量。

- 新增 `.text-up` / `.text-down` 全局颜色类
